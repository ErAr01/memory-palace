# Telegram Chat Indexing Architecture

## Overview

Бот индексирует сообщения из Telegram чатов для поиска объявлений о продаже.

## Выводы из отладки

### Ограничения Telegram API

1. **Активность чатов**: Популярные чаты (baraholka_tbi) могут иметь 10,000+ сообщений за 2 недели
2. **Медиа-сообщения**: `message.text` = `None` для сообщений с медиа (фото/видео). Текст подписи доступен через `message.raw_text` или `message.message`
3. **Лимиты API**: Telethon `iter_messages` с `limit=None` получает все сообщения, но период должен быть ограничен `from_date`/`to_date`

### Стратегия лимитов

- **НЕ использовать**: `limit` параметр для ограничения количества сообщений
- **Использовать**: `from_date` и `to_date` для ограничения периода (максимум 14 дней)

## Архитектура

### Дефолтные чаты

Чаты из `config/chats.yaml`:
- baraholka_tbi
- avito_baraholka_tbilisi
- baraholka_tbilisi_home

**Поведение**:
1. При первом запуске: индексация 2 недель истории
2. После этого: инкрементальное обновление каждый час после запроса пользователя
3. `get_missing_ranges()` вычисляет только недостающие периоды

### Кастомные чаты

Чаты, запрошенные пользователем, но не в дефолтном списке.

**Поведение**:
1. При запросе к непроиндексированному чату → предупреждение
2. Пользователь подтверждает → единоразовая полная индексация (14 дней)
3. Данные не обновляются автоматически (только по повторному запросу с force)

### Очистка данных

pg_cron ежедневно удаляет сообщения старше 14 дней.

## Компоненты

### TelegramIndexer (`src/indexer/client.py`)

```python
async def fetch_messages(
    chat: ChatIdentifier,
    from_date: datetime | None = None,
    to_date: datetime | None = None,
    limit: int | None = None,  # None = без лимита
) -> list[dict]
```

**Важно**: Период ограничивается через `from_date`/`to_date`, не через `limit`.

### IndexingService (`src/indexer/service.py`)

```python
async def index_chats(
    chats: list[ChatIdentifier],
    days: int = 14,
    force: bool = False,  # True = игнорировать кэш
) -> dict[str, int]
```

**Флаг `force`**:
- `False`: использует `get_missing_ranges()` для инкрементального обновления
- `True`: индексирует весь запрошенный период заново

### SearchAgent (`src/ai/agent.py`)

Проверяет кастомные чаты перед поиском:

```python
async def search(user_message: str) -> SearchResult:
    # 1. Парсинг запроса
    # 2. Проверка кастомных чатов на индексацию
    # 3. Если не проиндексированы -> need_indexing=True
    # 4. Иначе -> поиск
```

## Конфигурация (`config/chats.yaml`)

```yaml
default_days: 14      # Период индексации по умолчанию
max_days: 14          # Максимальный период поиска
index_cache_minutes: 60  # Интервал обновления индекса
```
